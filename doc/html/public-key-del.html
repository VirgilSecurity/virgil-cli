NAME

public-key-del −− delete a Public Key.

SYNOPSIS

virgil public-key-del [−−public-key-id arg] [−−confirm bool] [−−private-key file] [−−private-pwd arg]

DESCRIPTION

The purpose is to remove a Public Keyâs data. Note:

If -c, --confirm - true is used, the Public Key will be removed immediately without anyconfirmation.

If -c, --confirm - false is used, confirmation is required. The action will return an action_token response object and will send confirmation tokens to all of the Public Keyâs confirmed UDIDs. The list of masked UDIDâs will be returned in user_ids response object property. To commit a Public Key remove call ’user-data-confirm’ action with action_token value and the list of confirmation codes.

OPTIONS

−e arg, −−public-key-id arg Virgil Public Key.

Format:[public-id|email|phone|domain]:<value>
where:

•

if public-id, then − user's public-key-id;

•

if email, then − user's email;

•

if phone, then − user's phone;

•

if domain, then − user's domain.

−c, −−confirm bool

Default false. If -c, --confirm - false is used, the Public Key confirmation is required. If -c, --confirm - true is used, the Public Key will be removed immediately without anyconfirmation, but you must to enter Private Key. And Private Key Password if you have.

−k file, −−private-key file

Sender's private key.

−p arg, −−private-pwd arg

Sender's private key password.
